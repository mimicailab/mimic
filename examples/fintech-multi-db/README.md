# Fintech Multi-DB Example -- PostgreSQL + MongoDB

The flagship example of mimic's multi-database capabilities. This demonstrates a
personal finance and banking platform where **structured financial data** lives in
PostgreSQL and **unstructured activity/preference data** lives in MongoDB -- with a
single AI agent querying both transparently.

## Architecture

```
                          +---------------------+
                          |    AI Agent (:3004)  |
                          |  OpenAI Agents SDK   |
                          |  + Claude Haiku      |
                          +----------+----------+
                                     |
                    +----------------+----------------+
                    |                                 |
              MCP (stdio)                      Direct Driver
                    |                                 |
           +--------+--------+             +----------+----------+
           |   PostgreSQL    |             |      MongoDB        |
           |   (port 5432)  |             |   (port 27017)      |
           |                 |             |                     |
           |  users          |             |  activity_logs      |
           |  accounts       |             |  user_preferences   |
           |  transactions   |             |  notifications      |
           +-----------------+             +---------------------+

   Structured financial data           Semi-structured documents
   (balances, transactions,            (activity events, settings,
    account details)                    alerts, feature flags)
```

**How it works:** The agent uses the [OpenAI Agents JS SDK](https://github.com/openai/openai-agents-js)
with Claude as the model. It spawns `mimic host --transport stdio` as a subprocess
for auto-discovered PostgreSQL MCP tools, and connects directly to MongoDB for
document-based tools. The AI model decides which database to query based on the
user's question.

## Prerequisites

- Node.js >= 22
- Docker and Docker Compose
- Anthropic API key (`ANTHROPIC_API_KEY`)

## Quick Start

```bash
# 1. Start both databases
docker compose up -d

# 2. Set environment variables
export DATABASE_URL="postgresql://mimic:mimic@localhost:5432/mimic_fintech"
export MONGO_URL="mongodb://localhost:27017"
export ANTHROPIC_API_KEY="your-key-here"

# 3. Generate synthetic data for both personas
mimic run

# 4. Seed BOTH databases in one command
mimic seed --verbose

# 5. Inspect what was generated
mimic inspect schema
mimic inspect data

# 6. Install agent dependencies and start
cd agent && npm install && npm start

# 7. Query PostgreSQL-backed data (transactions, accounts)
curl -s -X POST http://localhost:3004/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "What are my account balances?"}' | jq

# 8. Query MongoDB-backed data (activity, preferences)
curl -s -X POST http://localhost:3004/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Show my unread notifications"}' | jq

# 9. Cross-database question
curl -s -X POST http://localhost:3004/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "How much did I spend on dining last month, and do I have any alerts about it?"}' | jq
```

## What Gets Seeded

### PostgreSQL (structured financial data)

| Table          | Active Trader | Saver       |
|----------------|---------------|-------------|
| `users`        | 1 row         | 1 row       |
| `accounts`     | 5+ rows       | 2-3 rows    |
| `transactions` | 500+ rows     | 100+ rows   |

**Active Trader** persona: High transaction volume across checking, savings,
investment, and credit accounts at multiple institutions. Frequent stock trades,
dining, ride-shares, and subscription charges.

**Saver** persona: Minimal accounts, careful spending, regular savings deposits,
and few discretionary purchases.

### MongoDB (semi-structured documents)

| Collection          | Description                                              |
|---------------------|----------------------------------------------------------|
| `activity_logs`     | Login events, page views, transfers, settings changes    |
| `user_preferences`  | Notification settings, display prefs, budget goals       |
| `notifications`     | Alerts, warnings, promotions, informational messages     |

## Agent Tools

### Auto-discovered MCP tools (PostgreSQL)

These tools are auto-generated by `mimic host` from the PostgreSQL schema:

| Tool                       | Description                                      |
|----------------------------|--------------------------------------------------|
| `get_transactions`         | Query with date range, category, merchant filters|
| `get_accounts`             | List accounts, filter by type or institution     |
| `get_users`                | Get user profiles                                |
| `get_transactions_summary` | Spending summary with category grouping          |

### Custom tools (MongoDB)

These tools are defined directly in the agent code:

| Tool                    | Description                                         |
|-------------------------|-----------------------------------------------------|
| `get_activity_log`      | Query activity events with date and action filters  |
| `get_user_preferences`  | Get user preferences and settings document          |
| `get_notifications`     | List notifications with read/unread filtering       |

## Configuration

### Environment Variables

| Variable           | Default                                                  | Description               |
|--------------------|----------------------------------------------------------|---------------------------|
| `DATABASE_URL`     | `postgresql://mimic:mimic@localhost:5432/mimic_fintech`  | PostgreSQL connection URL |
| `MONGO_URL`        | `mongodb://localhost:27017`                              | MongoDB connection URL    |
| `MONGO_DB`         | `mimic_fintech`                                          | MongoDB database name     |
| `ANTHROPIC_API_KEY`| (required)                                               | Anthropic API key         |
| `PORT`             | `3004`                                                   | Agent HTTP port           |
| `MODEL`            | `claude-haiku-4-5`                                       | Claude model to use       |
| `MIMIC_CLI`        | (auto-resolved)                                          | Path to mimic CLI binary  |

### mimic.json

The configuration file defines both database targets. When you run `mimic seed`,
it seeds **both** PostgreSQL and MongoDB from the same generated data set, ensuring
consistent user IDs across databases.

## Multi-DB Seed Strategy

| Database   | Strategy               | Behavior                                  |
|------------|------------------------|-------------------------------------------|
| PostgreSQL | `truncate-and-insert`  | Truncates tables, inserts structured rows |
| MongoDB    | `delete-and-insert`    | Deletes documents, inserts new ones       |

Both strategies ensure idempotent seeding -- run `mimic seed` repeatedly and get
the same clean result.

## Cleanup

```bash
# Remove seeded data
mimic clean --yes

# Stop and remove containers + volumes
docker compose down -v
```
